## 💡 Tips & Troubleshooting

### Memory Issues

**CUDA Out of Memory:**
```bash
# Kurangi batch size
python train_denoiser.py --batch_size 8  # atau 4

# Atau gunakan SimpleCNN (lebih kecil)
python train_denoiser.py --model simple_cnn
```

**RAM Issues:**
```bash
# Kurangi num_workers di config.py
NUM_WORKERS = 2  # atau 0
```

---

### Training Sangat Lambat

**Pakai GPU!** Jika tidak punya:
1. **Google Colab** (Free): https://colab.research.google.com/
2. **Kaggle Notebooks** (Free): https://www.kaggle.com/
3. **Cloud services**: AWS, GCP, Azure (Berbayar)

---

### Loss Tidak Turun

1. **Cek learning rate:** Terlalu besar atau kecil?
   ```bash
   python train_denoiser.py --lr 0.0005  # atau 0.0001
   ```

2. **Cek data:** Pastikan clean-noisy paired dengan benar
   ```bash
   python xrd_dataset.py  # Test dataloader
   ```

3. **Coba model lain:**
   ```bash
   python train_denoiser.py --model simple_cnn  # jika pakai unet
   ```

---

### Loss Exploding (menjadi NaN)

1. **Lower learning rate:**
   ```bash
   python train_denoiser.py --lr 0.0001
   ```

2. **Add gradient clipping** (edit `train_denoiser.py`):
   ```python
   torch.nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
   ```

---

### Overfitting

**Gejala:** Train loss turun terus, tapi val loss naik

**Solusi:**
1. More data augmentation
2. Stronger regularization:
   ```bash
   python train_denoiser.py --weight_decay 1e-4  # increase
   ```
3. Early stopping (sudah built-in)

---

### Model Tidak Improve

**Stuck di local minimum?**

1. **Reset dengan learning rate lebih tinggi:**
   ```bash
   python train_denoiser.py --lr 0.005  # start fresh
   ```

2. **Try different architecture:**
   ```bash
   python train_denoiser.py --model unet  # jika pakai simple_cnn
   ```

3. **Change initialization:**
   Edit `models.py`, uncomment weight initialization
